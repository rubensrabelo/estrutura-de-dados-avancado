Resposta:

Eu encontrei um pequeno erro no pseudocódigo e corrigi a função para:

Função F(h)
    se h <= 1 então
        retorne h
    senão
        retorne F(h-1) + F(h-2)

Essa função é recursiva e, ao ser executada, gera uma árvore de Fibonacci, onde cada chamada da função corresponde a um nó da árvore.

Seja:

  • ( N(h) ) o número de nós da árvore de Fibonacci de altura (h)
  • ( C(h) ) o número de chamadas da função (F(h))

Vamos provar, por indução matemática, que:

N(h)=C(h)N(h) = C(h)N(h)=C(h)

Casos base

Para (h = 0), a função executa apenas (F(0)):

N(0)=1N(0) = 1N(0)=1

C(0)=1C(0) = 1C(0)=1

Para (h = 1):

N(1)=1N(1) = 1N(1)=1

C(1)=1C(1) = 1C(1)=1

Logo,

N(0)=C(0)eN(1)=C(1)N(0) = C(0) \quad \text{e} \quad N(1) = C(1)N(0)=C(0)eN(1)=C(1)

Passo de indução

Ao calcular (F(h)), ocorre:

  • 1 chamada da função atual
  • chamadas de (F(h-1))
  • chamadas de (F(h-2))

Portanto, o número de chamadas é:

C(h)=1+C(h−1)+C(h−2)C(h) = 1 + C(h-1) + C(h-2)C(h)=1+C(h−1)+C(h−2)

Na árvore de Fibonacci ocorre o mesmo:

  • 1 nó raiz
  • subárvore de altura (h-1)
  • subárvore de altura (h-2)

Logo, o número de nós é:

N(h)=1+N(h−1)+N(h−2)N(h) = 1 + N(h-1) + N(h-2)N(h)=1+N(h−1)+N(h−2)

Hipótese de indução

Suponha que seja verdadeiro para um valor (k):

N(k)=C(k)N(k) = C(k)N(k)=C(k)

e

N(k−1)=C(k−1)N(k-1) = C(k-1)N(k−1)=C(k−1)

Provar para (k+1)

Partindo da definição de nós:

N(k+1)=1+N(k)+N(k−1)N(k+1) = 1 + N(k) + N(k-1)N(k+1)=1+N(k)+N(k−1)

Substituindo a hipótese de indução:

N(k+1)=1+C(k)+C(k−1)N(k+1) = 1 + C(k) + C(k-1)N(k+1)=1+C(k)+C(k−1)

Pela definição de chamadas:

C(k+1)=1+C(k)+C(k−1)C(k+1) = 1 + C(k) + C(k-1)C(k+1)=1+C(k)+C(k−1)

Portanto,

N(k+1)=C(k+1)N(k+1) = C(k+1)N(k+1)=C(k+1)

Portanto, como os casos base são verdadeiros e o passo de indução foi demonstrado, conclui-se que:

N(h)=C(h)N(h) = C(h)N(h)=C(h)

para todo (h > 0), ou seja, o número de nós da árvore de Fibonacci de altura (h) é igual ao número de chamadas da função (F(h)).

Demonstração (Prova Direta)

Pela definição, uma árvore binária completa é aquela que apresenta a seguinte propriedade:

se vvv é um nó tal que alguma subárvore de vvv é vazia, então vvv se localiza no último ou no penúltimo nível da árvore.

Já uma árvore AVL é:

uma árvore binária TTT tal que, para todo nó vvv, as alturas de suas duas subárvores, esquerda e direita, diferem em módulo de até uma unidade, ou seja:

∣hD(v)−hE(v)∣≤1|h_D(v) - h_E(v)| \leq 1∣hD​(v)−hE​(v)∣≤1

Vamos analisar os possíveis casos de um nó vvv em uma árvore binária completa.

Caso 1: nó acima do penúltimo nível

Se o nó vvv não está no último nem no penúltimo nível, então, pela definição de árvore completa, ele não pode ter subárvore vazia.

Logo, ele possui duas subárvores com a mesma altura:

hE(v)=hD(v)h_E(v) = h_D(v)hE​(v)=hD​(v)

Assim,

∣hD(v)−hE(v)∣=∣h−h∣=0≤1|h_D(v) - h_E(v)| = |h - h| = 0 \leq 1∣hD​(v)−hE​(v)∣=∣h−h∣=0≤1

Caso 2: nó no penúltimo nível

Pela definição de árvore completa, um nó no penúltimo nível pode ter uma subárvore vazia e outra não.

Assim, as alturas possíveis são:

hE(v)=1ehD(v)=0h_E(v) = 1 \quad e \quad h_D(v) = 0hE​(v)=1ehD​(v)=0

ou

hE(v)=0ehD(v)=1h_E(v) = 0 \quad e \quad h_D(v) = 1hE​(v)=0ehD​(v)=1

Logo,

∣hD(v)−hE(v)∣=∣1−0∣=1≤1|h_D(v) - h_E(v)| = |1 - 0| = 1 \leq 1∣hD​(v)−hE​(v)∣=∣1−0∣=1≤1

Caso 3: nó no último nível

O nó não possui subárvores.

hE(v)=0h_E(v) = 0hE​(v)=0

hD(v)=0h_D(v) = 0hD​(v)=0

Em todos os casos possíveis, a diferença entre as alturas das subárvores esquerda e direita de qualquer nó vvv é menor ou igual a 1.

∣hD(v)−hE(v)∣≤1|h_D(v) - h_E(v)| \leq 1∣hD​(v)−hE​(v)∣≤1

#include <iostream>
#include <algorithm>
using namespace std;

struct Node {
    int value;
    Node *left;
    Node *right;
};

// Função auxiliar que retorna a altura
// Se não for AVL, retorna -1
int altura(Node *node) {
    if (node == NULL)
        return 0;

    int h_esq = altura(node->left);
    if (h_esq == -1) return -1;

    int h_dir = altura(node->right);
    if (h_dir == -1) return -1;

    if (abs(h_esq - h_dir) > 1)
        return -1;

    return 1 + max(h_esq, h_dir);
}

bool eh_avl(Node *node) {
    return altura(node) != -1;
}

Resposta:

Para provar que a altura é O(log⁡n)O(\log n)O(logn), vou mostrar que o número mínimo de nós de uma árvore 2-balanceada de altura hhh cresce exponencialmente.

Se o número de nós cresce exponencialmente, então a altura cresce logaritmicamente em função de nnn.

Número mínimo de nós

Seja N(h)N(h)N(h) o número mínimo de nós de uma árvore 2-balanceada de altura hhh.

Para obter o menor número de nós possível, a árvore deve estar o mais desbalanceada possível, mas ainda respeitando:

∣hD(v)−hE(v)∣≤2|h_D(v) - h_E(v)| \leq 2∣hD​(v)−hE​(v)∣≤2

  • uma subárvore tem altura h−1h-1h−1
  • a outra tem altura h−3h-3h−3

Logo, o número mínimo de nós satisfaz a recorrência:

N(h)=1+N(h−1)+N(h−3)N(h) = 1 + N(h-1) + N(h-3)N(h)=1+N(h−1)+N(h−3)

onde:

  • 1 é o nó raiz
  • N(h−1)N(h-1)N(h−1) é a subárvore maior
  • N(h−3)N(h-3)N(h−3) é a subárvore menor

Casos base
